Tailwind ist ein "Utility-First"-Framework. Das bedeutet, anstatt vordefinierte Komponenten (wie `btn` oder `card` in anderen Frameworks) zu haben, bietet es dir winzige, einzelne CSS-Eigenschaften als Klassen an.
Das Ziel ist es das man so gut wie kein CSS nutzen muss. Man baut seine UI direkt in seiner HTML Struktur, indem man es mit Utility-Klassen kombiniert.

Warum das gut ist:
- **Konsistenz:** Da du immer dieselben vordefinierten Werte verwendest (z.b "p-4" für Padding), sieht diene UI überall gleich aus.
- **Schnelle Entwicklung:** Du siehst sofort, was deine Klasse bewirkt, und kannst schnell experimentieren.
- **Geringe Dateigröße:** Unbenutztes CSS wird im Build-Prozess entfernt, was zu kleineren Bundles führt.
- **Wartbarkeit:** Änderungen sind lokal und beeinflussen nicht andere Teile der UI.
